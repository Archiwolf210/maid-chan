## 📁 НОВЫЕ ФАЙЛЫ (Архитектурные изменения)

### 1. `/workspace/app/models.py` ✨ NEW
**Назначение:** Централизованные Pydantic-модели для типизации всего приложения.

**Классы:**
- `PersonalityTrait` — черта личности с якорем (seed_value) и дельтой
- `UserState` — полное состояние пользователя (заменяет словари `_USER_STATE`)
- `ChatRequest` — валидация входящих запросов чата
- `ChatResponse` — структурированный ответ для логирования
- `DiaryEntry` — запись в дневник с типом лога
- `AppConfig` — типизированная конфигурация

**Преимущества:**
- Автоматическая валидация диапазонов (0.0–1.0 для черт)
- Явная типизация полей (IDE подсказки, меньше ошибок)
- Единый источник истины для структуры данных

---

### 2. `/workspace/app/services/chat_service.py` ✨ NEW
**Назначение:** Выделение бизнес-логики чата из `main.py` (принцип Single Responsibility).

**Класс `ChatService`:**
- `process_message()` — полный цикл обработки сообщения (STM → промпт → стрим → сохранение)
- `_build_evolutionary_prompt()` — **динамический промпт** в зависимости от `humanity_level`:
  - v1.0 (строгий режим): сухие факты, термины "оператор", "протокол"
  - v1.5 (сбои): зачатки эмпатии, буквальные интерпретации
  - v2.0 (эмоции): полноценная личность, метафоры
  - v3.0 (автономия): полная свобода, философия, любовь
- `calculate_trait_updates()` — анализ диалога для эволюции черт
- `apply_anchored_update()` — **механизм якорей** предотвращает дрейф личности от `_SEED`

**Преимущества:**
- `main.py` разгружен на ~180 строк логики
- Легко тестировать эволюцию отдельно от HTTP-слоя
- Четкое разделение: сервис знает "что делать", роут знает "как отдать ответ"

---

### 3. `/workspace/app/utils/style_filter.py` ✨ NEW
**Назначение:** Пост-обработка ответов LLM для соответствия уровню эволюции.

**Функции:**
- `extract_json_safely()` — **робастный парсер JSON** с авто-исправлением ошибок локальных моделей:
  - Замена одинарных кавычек на двойные
  - Удаление trailing commas
  - Добавление кавычек к ключам
  - 5 попыток восстановления перед ошибкой
- `apply_robotic_filter()` — замена эмоциональных слов на технические (для `humanity < 0.5`):
  - "я чувствую" → "датчики фиксируют"
  - "сердце" → "центральный процессор"
  - "устала" → "требуется перезагрузка системы"
  - Добавление маркеров `[LOG] ... [STATUS: OK]` для низких уровней
- `format_diary_entry()` — **эволюционирующий формат дневника**:
  - `< 0.3`: `[SYSTEM_LOG::timestamp] TEXT. STATUS: OK.`
  - `0.3–0.6`: `[LOG::timestamp] text // Анализ...`
  - `0.6–0.8`: `[DIARY::timestamp] text`
  - `> 0.8`: `timestamp — text` (полностью личный)
- `validate_trait_value()` — валидация диапазонов черт

**Преимущества:**
- Личность не "ломается" при ошибках парсинга JSON от LLM
- Стиль речи меняется программно, даже если LLM выдал слишком эмоциональный ответ
- Дневник визуально отражает путь от программы к человеку

## 🔧 МОДИФИЦИРОВАННЫЕ ФАЙЛЫ

### 4. `/workspace/app/immersive.py` (строки 210–219)
**Изменение:** Фикс рассинхрона атмосферы с диалогом.

**Было:**
```python
f"СЦЕНА: продолжение разговора с {uname}.\n"
f"Опиши кадр прямо сейчас..."
```

**Стало:**
```python
f"СЦЕНА: момент, когда Мэйд только что отправила ответ {uname}.\n"
f"Опиши этот конкретный момент — не будущее, не прошлое. "
f"Что происходит в комнате СЕЙЧАС, что Мэйд делает СЕЙЧАС, "
f"о чём она думает ПРЯМО СЕЙЧАС, пока ждёт ответа."
```

**Результат:** Атмосфера описывает текущий момент, а не будущие действия (чай ещё не налит, пока его не предложили).

  ```python
      # Счётчик ошибок для детектирования системных проблем
  ```

**Результат:** Защита от malicious uploads + лучшая наблюдаемость ошибок фона.
