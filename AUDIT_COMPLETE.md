## ⚠️ ОТКЛОНЁННЫЕ ПРЕДЛОЖЕНИЯ (ОБОСНОВАННО)

### 1. Разбиение main.py на подмодули
**Причина отклонения:**
- Попытка создания `app/chat.py` привела к циклическим импортам
- Текущая модульность достаточна для проекта такого масштаба
- Риск регрессий превышает пользу от рефакторинга
- Проект стабилен, архитектурные изменения без явной необходимости нежелательны

### 2. Унификация нейминга
**Причина отклонения:**
- Текущий стиль консистентен: `_` для приватных, без префикса для публичных, UPPER_CASE для констант
- Глобальная переименовка создаст риски без существенной пользы
- Конфликтов с соглашениями Python не выявлено

### 3. Введение Pydantic-моделей
**Причина отклонения:**
- Увеличит сложность кодовой базы
- Текущая работа с dict достаточно надёжна
- Для локального проекта избыточно
